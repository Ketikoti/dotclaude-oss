# Contributing to dotclaude-oss

Thank you for your interest in contributing! This project exists to help developers build better with Claude Code, and your contributions make it stronger for everyone.

## Ways to Contribute

- **Add a new rule** — Share a coding convention or standard that works well for your team
- **Add a new command** — Contribute a workflow command that improves developer productivity
- **Add a new agent** — Propose a specialized agent for a specific domain or task type
- **Add a new skill** — Document a multi-step workflow that others can reuse
- **Improve existing files** — Fix mistakes, clarify wording, or add missing scenarios
- **Report issues** — Open an issue if something is unclear, broken, or missing

## Before You Start

1. Check [open issues](../../issues) to avoid duplicating work
2. For major changes, open an issue first to discuss the approach
3. Keep contributions focused — one concern per PR

## Contribution Guidelines

### File Structure
- Rules go in `.claude/rules/`
- Commands go in `.claude/commands/`
- Agents go in `.claude/agents/`
- Skills go in `.claude/skills/`

### Writing Style
- Use clear, direct language — no fluff
- Use bullet points for lists of rules or steps
- Include a brief description at the top of each file
- Include a "Definition of Done" or "Output Format" section where relevant
- Keep files focused — one topic per file

### Commit Messages
Use [Conventional Commits](https://www.conventionalcommits.org/):
- `feat:` for new rules, commands, agents, or skills
- `fix:` for corrections to existing content
- `docs:` for README or documentation changes
- `refactor:` for restructuring existing content

## Pull Request Process

1. Fork the repository
2. Create a branch: `feat/your-contribution-name`
3. Make your changes
4. Ensure your files follow the existing format and style
5. Submit a PR with a clear description of what you added and why

## Code of Conduct

This project follows the [Contributor Covenant Code of Conduct](CODE_OF_CONDUCT.md). By participating, you agree to uphold it.

## Questions?

Open an issue or start a discussion — we're happy to help.
